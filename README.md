# 📦 Tomyris AI Dashboard - Тестовое задание

Полноценная интеграция RetailCRM, Supabase и Telegram-бота с красивым дашбордом на Next.js. Проект реализован с использованием AI-ассистента (Вайбкодинг).

## 🛠 Технологический стек
* **Фронтенд:** Next.js (React), Tailwind CSS, Recharts, Lucide React
* **Бэкенд/Скрипты:** Python (Requests, python-dotenv)
* **База данных:** Supabase (PostgreSQL)
* **Деплой:** Vercel

## 🚀 Архитектура проекта
Проект разделен на две независимые части:
1. **Фронтенд (`/dashboard`):** Веб-приложение, которое в реальном времени отображает статистику заказов из Supabase.
2. **Бэкенд-воркер (`main.py`):** Единый Python-скрипт, который объединяет логику:
   * Единожды загружает моковые данные (`mock_orders.json`) в RetailCRM.
   * Каждую минуту забирает новые заказы из RetailCRM и делает Upsert в Supabase.
   * Каждую минуту проверяет VIP-заказы (> 50,000 ₸) и отправляет алерты в Telegram (без дубликатов).

## 🤖 Опыт Вайбкодинга (AI-Assisted Development)
Проект был полностью написан в тандеме с AI (Gemini).

**Где застряли и как решили:**
1. **Проблема:** Ошибка `Order is not loaded` при первичной загрузке в RetailCRM. Система ругалась на несуществующие `orderType` (`eshop-individual`) и `orderMethod` (`shopping-cart`).
   * **Решение:** Проанализировали документацию и структуру демо-аккаунта. Решили проблему не переписыванием кода, а правильной настройкой самой CRM — добавили нужные символьные коды в справочники системы.
2. **Проблема:** Дублирование сообщений в Telegram при каждом запуске воркера.
   * **Решение:** AI предложил и внедрил систему кэширования: скрипт записывает ID уже обработанных заказов в `processed_orders.txt` и проверяет его перед отправкой следующего алерта.
3. **Проблема:** Дублирование записей в Supabase.
   * **Решение:** Настроили колонку `id` как Primary Key в самой БД и добавили HTTP-заголовок `Prefer: resolution=merge-duplicates` для реализации идемпотентности.
